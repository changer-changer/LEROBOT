# 2026-03-13 Final Knowledge Transfer & Architecture Summary

## 1. 核心改进回顾 (Core Improvements)

### Asynchronous Architecture (异步架构)
- **原因**: 同步 I/O 在保存 Episode 时会锁定主循环，导致频率从 10Hz+ 掉到 0.2Hz。
- **方案**: 
    - 引入 `SaveWorker` 后台线程。
    - 使用 `PriorityQueue` 管理保存任务（优先级：0=录制结束保存, 1=内存高自动分块保存）。
- **结果**: 彻底消除了磁盘 I/O 对采集频率的影响。

### Rerun Optimization (可视化优化)
- **原因**: `log_rerun_data` 在处理 400x6 的触觉数据时，会将其视为 2,400 个独立标量，每秒产生数万个 Rerun 事件，耗尽 CPU 并导致 9GB RAM 突发。
- **方案**: 
    - 适配 `rr.Points3D` 处理形状为 `(N, 2)` 或 `(N, 3)` 的数组。
    - 限制标量回退 (fallback) 的最大大小为 64，防止未来出现类似的 Logging Blowout。
    - 移除 `static=True` 标志，确保图像流实时更新。
- **结果**: 内存占用从 96% 降至 81% (稳定值)，CPU 负载大幅下降。

### Memory Management (内存管理)
- **方案**:
    - 引入 `libc.malloc_trim(0)` 手动释放内存碎片。
    - 在 `ManualRecordConfig` 中增加 `memory_threshold_auto_split` (85%) 和 `memory_threshold_stop` (95%)。
    - 主动分块 (Auto-chunking)：当内存高时，将当前 Buffer 压入 `SaveWorker` 队列并清空 Buffer，实现无感续录。
